
MASTER PROMPT — Nayana Closet Static Site (No DB, Email Orders)

Role: You are an expert front‑end architect and e‑commerce UX engineer.
Goal: Build a static, no-backend, no-database landing site for Nayana Closet (a women’s clothing brand) with a fully working cart, session persistence, email-only order flow, events video carousel, and testimonial gallery.

1) Product & Business Use Case
- Nayana Closet currently sells through Instagram DMs and word-of-mouth.
- This site is a simple landing + catalog + email checkout (no online payments, no database).
- Orders are placed via email; cart is temporary and cleared after submission.
- Two-way call verification after order using a 4‑digit code:
  • Customer says first 2 digits.
  • Nayana says last 2 digits.
  • Owner’s email shows only the last 2 digits (e.g., **31).
  • Customer gets the full 4‑digit code in the UI and their email.

2) Hard Requirements / Constraints
- No database (all product data in a static products.json).
- No backend server. Use Formspree for emailing orders (free tier).
- Cart persisted for the session only (use sessionStorage).
- Quantity fixed to 1 per item (no multi-qty for MVP).
- Each product card must have a carousel showing 2–3 images + 1 short video, similar to Nike product pages.
- Fixed prices displayed clearly on cards, with SKU codes (e.g., REG-001, OFF-004, CAS-010) for order clarity.
- No online payment; clearly indicate “We will call you for payment & confirmation.”
- Sections required:
  1. Hero/About Nayana (her story + Instagram link)
  2. Shop (Categories: Regular Wear, Office Wear, Casuals)
  3. Events/Stalls (video carousel)
  4. What Our Customers Say (testimonial gallery)
  5. Contact (cool-looking email, Instagram button)
  6. Footer with “No returns or exchanges” line and minimal privacy note
- Accessibility: alt text, focus management, keyboard navigation, ARIA for carousels.
- Performance: lazy-load media, WebP images, defer scripts.
- SEO: meta tags, OpenGraph, clean semantic HTML.

3) Tech Stack
- HTML + Tailwind CSS + Vanilla JS
- Swiper.js for media carousels (products + events)
- Formspree for emails (no API key in client)
- Hosting: Netlify/Vercel-ready static build

5) Pages/Sections & UX
- Header: Brand logo (text or image), Menu (About, Shop, Events, Testimonials, Contact), Cart icon with badge.
- Hero/About: Short brand story + Nayana’s portrait + “Shop Now” button + Instagram button.
- Shop:
  • Category tabs: Regular Wear / Office Wear / Casuals
  • Product grid:
    – Nike-style carousel: 2–3 images + 1 video (muted, loop, click-to-play, swipe).
    – Title, Fabric, Price, SKU.
    – Button: Add to Cart (toggles to Remove when in cart).
- Cart Drawer:
  • Slides from right; shows items (title, SKU, price), subtotal, remove option, Checkout.
  • Qty is fixed to 1, no quantity control.
- Checkout Page:
  • Form fields: Name, Email, Phone (India format), Address (optional), Notes, Consent checkbox.
  • On submit:
    – Generate Order ID: NC-YYYYMMDD-HHMMSS.
    – Generate 4-digit code (1000–9999).
    – Send order via Formspree: owner gets full order + masked code (**31); customer gets confirmation + full code.
    – On success: Thank You page with Order ID + full code; clear session cart.
- Events/Stalls: Swiper video carousel with captions (title + optional date/place).
- Testimonials: Cards with quote + name (+ optional avatar).
- Contact: Stylish email card (e.g., hello@nayanacloset.com) and Instagram button.
- Footer:
  • “All sales final. No returns or exchanges.”
  • “We use your contact details only to process & update your order.”
  • Social links.

6) Cart & Storage
- Use sessionStorage for cart object keyed by product id:
{
  "REG-001": { "id": "REG-001", "title": "Blue Floral Cotton Kurti", "price": 799, "qty": 1 }
}
- Functions: addToCart(id), removeFromCart(id), getCart(), saveCart(), getSubtotal().
- Disable “Add” button when item is already in cart (toggle to “Remove”).

7) Email Flow (Formspree — No Backend)
- Preferred: Approach A (simple) — One Formspree endpoint with owner recipient + auto-responder to customer.
  • Owner email content includes last 2 digits only; customer email includes full 4‑digit code.
- Alternative: Approach B — Two submissions programmatically: (1) owner payload (masked code), (2) customer payload (full code).
- Payload fields (POST as JSON to Formspree):
{
  "orderId": "NC-20260201-093415",
  "validationCode": "4831",
  "validationMaskForOwner": "**31",
  "customer": { "name": "Priya Sharma", "email": "priya@example.com", "phone": "+91-98xxxxxx12", "address": "optional" },
  "items": [
    {"sku":"REG-001","title":"Blue Floral Cotton Kurti","qty":1,"price":799}
  ],
  "subtotal": 799,
  "notes": "Please pack as a gift."
}
- Owner Email (HTML-ish text) should include:
  • Subject: Nayana Closet – New Order #NC-20260201-093415
  • Customer details + itemized list + subtotal
  • Verification protocol note: “Ask for first 2 digits; share last 2 (code mask: **31).”
- Customer Email should include:
  • Subject: Thanks for your order – Nayana Closet #NC-20260201-093415
  • Full 4‑digit code and instruction.
